No 1 About Project

This is an online multiplayer adventure game, which two players can cooperate across the internet to solve puzzles in Egypt.

Players will be in the main menu, then one player enter a server name and click 'create server', the other player can join session by searching that name. After that, two players are able to start their adventure and work together in the Egypt map.
At first, they can easily see a spinning key in front of them. Collect it, then it will appear on one of the columns. That's the hint which incidates we need to collect all keys for each column as key holder, then some mechanism will be triggered, so that we can reach that shining treasure place where can only see through a fence at first. Then there are 3 levels to be solved in order to get corresponding keys, and one more free key hidden in the map.
lv1 is designed to use objects nearby to suppress traps on the ground, one player stands on the pressure plate and door automatically open for the other player to get the key. lv2 also needs two players to cooperate, activate and step onto the moving platforms alternatively to reach the key high up on the pyramid. lv3 will additionally require some jumping skills and thinking to find proper way to the key.

No 3 Features

—Online Subsystem Steam

Adopted GameInstance Subsytem to build a reusable subsysteman for multiple players to create and join online sessions. Configured with Steam platform service. Coded in C++ for create, find and join session functionality. Configured session setting and searching setting. Used dynamic multicast delegate to hook up those async functions. Also built an UI for this subsystem with button click spamming prevented.

—Various gameplay elements

Coded in C++ to create many classes for player to interact, like pressure plates, movable doors, floating platforms, place holder props, spinning keys with sound hint when collecting...and so on~

—Delegate

Utilized customized dynamic multicast delegates to implement game elements which can be triggered and cause multiple effects, making interactions of game elements more complex and powerful.

—RPC

Carefully managed replication for variables, movement and game elements creation and destruction to ensure synchronicity for all players. Leveraged NetMulticast RPC to deliver win area message and bring up win screen for all players.

—Customized player movement

Calculated and adjusted player movement by lots of tests on gameplay. For instance, in lv2, player is required to jump onto a floating platform with high speed, which needs prediction on jumping timing and angle. Polished collision for those critical parts of the map to jump onto and those objects need to be pushed. Ensured levels are challenging, achieveable and of great fun.
